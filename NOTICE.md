# Third-party data attribution

This package's country/subdivision datasets under `data/` are generated by
`scripts/build-data.mjs` from the following sources:

## iso-3166 (build-time only)

- Authoritative source of the **complete, current ISO 3166-1 / ISO 3166-2 subdivision set and
  codes**, i.e. which countries/subdivisions exist and their full codes (`ES-AN`) in `data/`.
  Also provides the canonical base name for subdivisions esosedi does not cover.
- https://github.com/wooorm/iso-3166 (MIT License)
- Used only at build time (listed under `devDependencies`); no runtime dependency is shipped.

## esosedi/3166 (`iso3166-2-db`)

- Source of the **localized name strings** (12 languages) in `data/*-localized.json`, plus the
  English base name for the subdivisions it covers (top-level regions).
- https://github.com/esosedi/3166 (MIT License)
- Copyright (c) 2016 esosedi. The upstream license text is retained at
  `scripts/vendor/LICENSE-esosedi-3166`.

Only the 12 fully-populated language files (en, ru, de, fr, es, zh, hi, pt, ja, ar, it, he)
are used. The upstream repo also contains stub locale files (am, bi, bn, da, kr) that are
essentially empty; these are intentionally excluded. A small number of upstream entries with
cross-script contamination (e.g. Cyrillic placeholders in the Chinese file) are dropped by the
generator, so consumers fall back to the English name.

Reconciliation of codes to official ISO 3166-2 (and any non-ISO regions dropped in the process)
is recorded in `data/reconciliation-report.json`.
